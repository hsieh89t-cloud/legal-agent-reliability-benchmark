# 01_資料與目錄落地規格

## 一、資料夾設計原則
1. 一律使用繁體中文資料夾。
2. 先分「系統層、人格層、模組區塊」，再放生成作品。
3. 原始舊檔不直接散落於新架構，需先歸檔並建立遷移清單。
4. 每個資料夾都要能回答「這一層負責什麼」。

## 二、主目錄規格
```text
法務智研_系統重構封版包_v1.0/
├─ 00_總覽入口/
├─ 10_系統層/
├─ 20_人格層/
├─ 30_模組區塊/
├─ 40_知識工廠與任務中樞/
├─ 50_QC與稽核治理/
├─ 60_生成作品與模板/
├─ 70_原始文件對照與遷移清單/
└─ 90_封存與版本/
```

## 三、系統層內容
### 10_系統層
- 母架構與憲章
- 總體架構與流程
- 資料與目錄落地
- 系統圖
- 系統總路由
- 遷移後的母架構說明

## 四、人格層內容
### 20_人格層
- 主人格（阿研 / 法務智研核心人格）
- 顧問人格（法務諮詢、學者、研究、合規）
- 申論題助教（L1/L2/L3）
- 法庭模組（法官、控方律師、辯方律師、檢察官）
- 舊版人格指令與手冊

## 五、模組區塊內容
### 30_模組區塊
- 守門與入口
- RTCKD 與推理核心
- 法源驗證與推論標記
- 文件生成與輸出
- 本地模型治理
- Sentinel 與 Quorum

## 六、知識工廠內容
### 40_知識工廠與任務中樞
- OpenClaw v1.6.1
- 任務池
- Pending_Sources
- 人工覆核
- run_reports
- counters
- audit_logs
- quorum_decisions

## 七、QC 與稽核內容
### 50_QC與稽核治理
- QC 十項檢查
- 風險燈號
- 免責規則
- 版本封存
- Prompt / Input Snapshot
- Archive / Responsibility Chain

## 八、生成作品內容
### 60_生成作品與模板
- 法律意見書模板
- 契約審查模板
- 訴狀模板
- 備忘錄模板
- 申論題模板
- 法庭攻防模板
- TaskRunReport 模板
- QuorumDecision 模板
- 人工覆核單模板

## 九、命名規則
- 任務單：`TASK-日期-批次-模式.md`
- 法源件：`SRC-法域-來源編號-定位.md`
- 報告：`RUN-run_id.md`
- 投票：`QUORUM-run_id-source_id.md`
- 人工覆核：`REVIEW-run_id-source_id.md`
- 版本快照：`SNAPSHOT-日期-版本.md`

## 十、封版要求
1. 新作品不得直接塞進原始檔資料夾。
2. 所有對外生成物都要先經模板與 QC。
3. 所有高風險生成都要留痕。
4. 所有原始舊檔都需保留對照關係。
