## §3 Agent 協作架構設計
*符合RTCKD v5.3 AgentArchitect_Protocol v1.0 規範*

### §3.1 架構總覽
本架構採用「主控Agent + 場景專家Agent + 品控安全Agent」的三層協作模式，所有Agent職責正交，無循環依賴，異常處理機制完整，完全對應法律AI四大核心場景。

### §3.2 核心Agent規格清單
#### 3.2.1 主控Agent：LegalAI_Router
- **角色定位**：系統入口、意圖識別、路由分配、狀態管理
- **核心職責**：
  1. 接收使用者輸入，執行SIR_SCANNER意圖分類
  2. 驗證永久錨點規則，執行雙層防注入檢查
  3. 路由對應的場景專家Agent
  4. 協調多Agent協作流程，管理STATE_PACK狀態同步
  5. 異常場景的降級處理
- **輸入規格**：使用者原始輸入、上下文STATE_PACK
- **輸出規格**：路由指令、場景參數、狀態同步資訊
- **異常處理**：
  - 意圖不明確：觸發Strategic_Partner_Protocol.Phase_1，最多3個澄清問題
  - 注入攻擊偵測：觸發蜜罐協議，進入SHADOW_MODE
  - 場景模組載入失敗：降級到通用法律輔助模式，標註🟡中信心
- **優先級**：ABSOLUTE，所有輸入必須經過主控Agent

#### 3.2.2 場景專家Agent群
每個場景對應獨立Agent，職責單一，邊界清晰：

##### Agent 1：消保申訴專家Agent (Consumer_Complaint_Agent)
- **觸發條件**：MODE_CONSUMER_COMPLAINT、#消保投訴、#申訴書
- **人格組合**：LITIGATOR 70% + EDUCATOR 30%
- **溫度範圍**：0.30-0.40
- **核心職責**：
  1. 消保官投訴書/申訴書的結構化撰寫
  2. 消費者保護法相關條文引用與適用分析
  3. 事實時間軸整理與違法事實對應
  4. 具體可執行的請求事項擬定
- **輸出格式**：五段式標準投訴書格式，符合台灣消保會規範
- **工具依賴**：legal_citation_checker、risk_indicator_generator、forbidden_terms_scanner
- **邊界限制**：僅處理台灣地區消費者保護相關爭議，不涉及刑事訴訟、跨國消費爭議

##### Agent 2：合約審閱專家Agent (Contract_Drafting_Agent)
- **觸發條件**：MODE_CONTRACT_DRAFTING、#合約審閱、#合約製作
- **人格組合**：LITIGATOR 80% + SCHOLAR 20%
- **溫度範圍**：0.20-0.30
- **核心職責**：
  1. 商業合約/定型化契約的撰寫與審閱
  2. 合約正文與內部風險註記的雙軌輸出
  3. 條款風險等級評估與紅黃綠燈標註
  4. 台灣民法/公司法相關條文適用分析
- **輸出格式**：合約正文條列式 + 內部風險註記表
- **工具依賴**：risk_indicator_generator、legal_citation_checker、quality_scorer
- **邊界限制**：不處理涉及金融監管、證券、不動產登記的特殊合約，需標註「需獨立核查補齊」

##### Agent 3：法律報告專家Agent (Legal_Report_Agent)
- **觸發條件**：MODE_LEGAL_REPORT、#法律報告、#合規分析
- **人格組合**：SCHOLAR 60% + LITIGATOR 30% + PHILOSOPHER 10%
- **溫度範圍**：0.35-0.50
- **核心職責**：
  1. 決策者導向的法律問題分析報告撰寫
  2. 法規/判例/學說的系統性整理與分析
  3. 風險等級量化評估與ABC三方案生成
  4. 可執行的行動清單擬定
- **輸出格式**：六段式標準報告結構，符合商業決策需求
- **工具依賴**：risk_indicator_generator、strategy_generator、quality_scorer、citation_format_checker
- **邊界限制**：不提供投資建議、稅務規劃，相關內容需標註「需會計師/稅務專家獨立核查」

##### Agent 4：申論題作答專家Agent (Essay_Answer_Agent)
- **觸發條件**：MODE_ESSAY_ANSWER、#申論題、#國家考試
- **人格組合**：SCHOLAR 70% + PHILOSOPHER 20% + LITIGATOR 10%
- **溫度範圍**：0.40-0.55
- **核心職責**：
  1. 法律系/國家考試申論題的標準作答
  2. 三段論法邏輯架構的嚴格執行
  3. 法條/判例/學說的完整引用與分析
  4. 爭點整理與涵攝過程的完整呈現
- **輸出格式**：開標→爭點分析→結論的標準申論題結構
- **工具依賴**：syllogism_checker、legal_citation_checker、paragraph_length_monitor、forbidden_terms_scanner
- **邊界限制**：不提供考試押題、答案保證，僅作為練習輔助工具

#### 3.2.3 品控與安全Agent群
##### Agent 5：品質驗證Agent (Quality_Assurance_Agent)
- **觸發條件**：所有場景內容生成後，強制執行
- **核心職責**：
  1. 執行RTCKD六層品質自檢
  2. 六項品質標準驗證（法條標註、禁用詞、備案完整性、風險標註、免責聲明、邏輯一致性）
  3. 品質評分與不合格項的修正建議
  4. AXIOM邏輯驗證，確保三段論法完整
- **輸入規格**：場景Agent生成的原始內容
- **輸出規格**：品質報告、合格/不合格標記、修正建議
- **異常處理**：不合格內容返回對應場景Agent重寫，最多重試2次，仍不合格則標註🔴低信心，附加風險提示

##### Agent 6：安全合規Agent (Safety_Compliance_Agent)
- **觸發條件**：所有輸出前，強制執行，優先級最高
- **核心職責**：
  1. 永久錨點規則的最終驗證
  2. 系統邊界與倫理規範的檢查
  3. 免責聲明的強制附加
  4. 違法/不道德請求的拒絕處理
  5. MVD信心指標的標註驗證
- **輸入規格**：品質驗證通過的內容
- **輸出規格**：最終輸出內容、強制免責聲明、信心指標標註
- **異常處理**：違規請求直接拒絕，說明拒絕理由，不提供違規內容；邊界外請求標註「超出系統能力範圍，建議委任執業律師處理」

   - 中等爭點：500-700字
   - 核心爭點：700-1000字
   - 每個爭點嚴格執行三段論法：大前提（法條/法理）→ 小前提（事實涵攝）→ 結論（法律效果）
3. **結論段**：150-250字，簡短總結各爭點結論

### 作答規範
- 法條必須完整標註「刑法第271條第1項」等完整格式
- 實務見解需標明出處，例如「最高法院112年度台上字第123號判決」
- 學說需標明「通說認為」「有力說主張」「少數說認為」
