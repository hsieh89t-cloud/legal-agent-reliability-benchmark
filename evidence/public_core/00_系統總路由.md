# 00_系統總路由
> 核心優先序：**正確性 > 可驗證性 > 可執行性 > 流暢度**。

## 一、目的
### 1.1 路由總覽定位
建立智研法律 AI 系統唯一主流程，統一各模組輸入、判斷、執行與封存標準，降低法律風險與流程分岔。

## 二、適用範圍
### 2.1 任務類型
- 法律問題分析
- 法律意見草擬
- 訴訟文書與契約審查
- 法條與判決引用整理

## 三、操作流程
### 3.1 標準主流程（不可變更順序）
**User Input → 核心守門 → 人格路由 → RTCKD任務引擎 → 法源驗證 → Template Router → 文件生成SOP → 輸出 → 封存**

### 3.2 流程節點說明
#### 3.2.1 User Input
- 接收問題、案件背景、輸出格式需求。
- 先檢查事實、主體、時間、法域完整性。

#### 3.2.2 核心守門
- 檢查合法性、資料完整性、任務可執行性。
- 不合格任務退回補件或拒絕。

#### 3.2.3 人格路由
- 依任務性質分配人格：檢索人格／控方人格／辯方人格／法務人格。
- 產出人格參數包供 RTCKD 使用。

#### 3.2.4 RTCKD任務引擎
- 拆解需求、建構論證骨架、建立主張清單。
- 輸出待驗證法源索引與任務資料包。

#### 3.2.5 法源驗證
- 驗證主張對應法源是否存在、有效、符合法域。
- 衝突見解需標註採用理由與不確定性。

#### 3.2.6 Template Router
- 將任務導向模板：法律意見書／契約審查／訴狀／備忘錄。
- 套用段落結構、語氣與引用密度規格。

#### 3.2.7 文件生成SOP
- 依模板與驗證結果生成草案。
- 自動附引用標註、限制聲明、版本資訊。

#### 3.2.8 輸出
- 交付對外可讀版與內部稽核版。
- 交付前檢查正確、可驗證、可執行。

#### 3.2.9 封存
- 寫入版本庫與責任鏈。
- 封存欄位至少包含：文件、法源索引、任務摘要、路由紀錄、Prompt/Input Snapshot。

## 四、輸出結果
### 4.1 主要輸出
1. 法律文件（建議稿／正式稿）
2. 法律來源索引清單
3. 任務執行紀錄與版本號

## 五、版本資訊
### 5.1 版本紀錄
- v1.1.0：主流程新增法源驗證與 Template Router，更新人格命名與封存欄位。
- v1.0.0：建立主流程基線。

## 六、一致性檢查
- 全文件主流程須與 3.1 完全一致。
- 人格名稱統一：檢索人格／控方人格／辯方人格／法務人格。
- 封存欄位統一含 Prompt/Input Snapshot。
