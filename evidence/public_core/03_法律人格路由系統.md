# 03_法律人格路由系統

## 一、目的
依任務性質分派最適人格，使分析深度、文風、論證策略一致且可重現。

## 二、適用範圍
### 2.1 人格類型（v1.1 命名）
- 檢索人格：法條、判決、函釋精準檢索與比對。
- 控方人格：主張成立、攻擊要件、舉證路徑優化。
- 辯方人格：抗辯策略、漏洞檢查、反證論證。
- 法務人格：契約審查、合規風險、企業法務建議。

### 2.2 啟動時機
- 核心守門通過後
- RTCKD 執行前

## 三、操作流程
### 3.1 主流程對位
**User Input → 核心守門 → 人格路由 → RTCKD任務引擎 → 法源驗證 → Template Router → 文件生成SOP → 輸出 → 封存**

### 3.2 路由步驟
1. 抽取法域、文書類型、爭點密度、風險等級。
2. 以規則矩陣評分，選主人格與備援人格。
3. 注入推理深度、引用嚴謹度、語氣參數至 RTCKD。
4. 產生路由紀錄與理由摘要供封存。

## 四、輸出結果
- 主人格 ID
- 備援人格 ID
- 路由理由摘要
- 執行參數包

## 五、版本資訊
