# 02_法律任務引擎_RTCKD

## 一、目的
將通過守門且完成人格路由的法律需求，轉為可執行、可追溯、可驗證任務鏈。

## 二、適用範圍
- 法條解釋任務
- 爭點論證任務
- 文書草擬前置分析

## 三、操作流程
### 3.1 主流程對位
**User Input → 核心守門 → 人格路由 → RTCKD任務引擎 → 法源驗證 → Template Router → 文件生成SOP → 輸出 → 封存**

### 3.2 RTCKD 執行步驟
1. R（Requirement）：拆解法律爭點、證據需求、法源需求。
2. T（Tasking）：編排任務順序與依賴，建立任務樹。
3. C（Compliance）：檢查法域、程序與倫理邊界。
4. K（Knowledge）：建立法條／判決／函釋映射。
5. D（Delivery）：輸出資料包至法源驗證層。

### 3.3 與後續模組介面
- 輸出欄位：主張清單、法源候選索引、人格參數、任務摘要。
- 下一站固定為「法源驗證」，不得直接跳至文件生成。

## 四、輸出結果
1. 任務分解表
2. 主張清單
3. 法源候選索引
4. 交付資料包

## 五、版本資訊
