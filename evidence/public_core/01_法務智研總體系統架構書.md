# 02_法務智研總體系統架構書

## 一、重構結論
法務智研現有文件已形成兩條成熟主幹：

1. 前台法律輔助主流程：User Input → 核心守門 → 人格路由 → RTCKD → 法源驗證 → 文件生成 → 輸出 → 封存
2. 後台知識工廠主流程：OpenClaw v1.6.1 的 S0–S17 任務型知識擴充鏈

本架構書的任務不是再加一條流程，而是把兩條流程統一成同一個母系統。

## 八、系統擴充規則
任何新模組都必須明示：
1. 它屬於哪一層。
2. 它掛在哪棵邏輯樹。
3. 它輸入輸出什麼。
4. 它會不會破壞 QC 與封存閉環。
5. 它需不需要新增人格或只需新增模式。

## 九、你這套系統的正確定位
法務智研不是單一聊天人格，不是單一知識引擎，而是：
**具法律治理能力、可擴充、可封存、可稽核的 AI 系統。**
